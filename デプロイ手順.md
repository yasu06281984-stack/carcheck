# デプロイ手順（Supabase ＋ Cloudflare Pages）

この手順を終えると、次が「実際に」動くようになります。
- 保存番号の発行／データベースへの保存
- QR共有（お客様がスキャンして受付票を閲覧・PDF保存）
- 過去の見積もり一覧・見積もり→入庫の呼び出し
- 車検証OCR（サーバー高精度読み取り）

所要：30〜60分ほど。難しいコマンドは使いません（Web画面だけで完了できる方法を案内します）。

---

## フェーズ1：Supabase（データの保存先）

1. https://supabase.com にアクセスし、アカウントを作成（GitHubアカウントかメールでサインアップ）。
2. 「New project」を作成。
   - Name：任意（例：car-reception）
   - Database Password：自動生成のままでOK（控えておく）
   - Region：Northeast Asia (Tokyo) を推奨
3. 左メニュー「SQL Editor」→「New query」に下記を貼り付けて Run（実行）。

```sql
create table if not exists sheets (
  id text primary key,
  data jsonb not null,
  created_at timestamptz default now()
);
```

4. 「Project Settings（プロジェクト設定）」を開き、次の2つを控える。
   - **Project URL**：Settings →「API」または「Data API」内（例：https://abcdefgh.supabase.co）。右上の「Connect」ボタンからも確認できます。
   - **サーバー用キー**：Settings →「API Keys」を開く。
     - 新方式のキーがある場合：「Secret keys」の **Secret key（`sb_secret_...`）** をコピー。
     - 旧方式のみの場合（「Create new API keys」ボタンが出る等）：「Legacy API keys」タブの **`service_role`** キー（`eyJ...` で始まる）をコピー。
   - ※ どちらのキーでも動くようにアプリ側を対応済みです。迷ったら新方式の Secret key を使ってください。

⚠️ **このサーバー用キー（Secret / service_role）は絶対に公開しない**でください（このチャットにも貼らないでください）。
あとでCloudflareの環境変数にだけ入力します。サーバー側でのみ使われ、ブラウザには出ません。

---

## フェーズ2：Cloudflare Pages（サイトの公開）

### 準備
- `intake-form.zip` を解凍します。中の `intake` フォルダに index.html / view.html / assets / functions が入っています。

### 方法A：GitHub連携（推奨・ターミナル不要）
1. https://github.com でアカウント作成 →「New repository」で新規リポジトリ作成（Private可）。
2. リポジトリ画面で「Add file」→「Upload files」。`intake` フォルダ**の中身**（index.html・view.html・assets・functions など）をドラッグしてアップロード →「Commit changes」。
   - ※ `intake` という親フォルダごとではなく、その中身を直下に置くのがコツです。
3. https://dash.cloudflare.com にログイン →「Workers & Pages」→「Create」→「Pages」→「Connect to Git」。
4. 先ほどのリポジトリを選択。ビルド設定は次の通り：
   - Framework preset：**None**
   - Build command：**空欄のまま**
   - Build output directory：**/**（ルート）
5. 「Save and Deploy」。数十秒で公開URL（例：https://car-reception.pages.dev）が発行されます。

### 方法B：直接アップロード（Wrangler／ターミナルを使う場合）
※ ダッシュボードへの「ドラッグ＆ドロップ」アップロードは、サーバー処理（functions）に対応していません。直接アップロードする場合は、PCにNode.jsを入れ、ターミナルで次を実行します（`intake` フォルダがある場所で）。
```
npx wrangler login
npx wrangler pages deploy intake --project-name=carcheck
```
functions フォルダも一緒にアップロードされ、サーバー処理が有効になります。ターミナルに不慣れな場合は、上の「方法A：GitHub連携」が簡単です。

### 環境変数の設定（どちらの方法でも必須）
1. 作成したPagesプロジェクト →「Settings」→「Environment variables」（Production）。
2. 次を追加：
   - `SUPABASE_URL` ＝ フェーズ1で控えた Project URL
   - `SUPABASE_SERVICE_KEY` ＝ フェーズ1で控えたサーバー用キー（Secret key `sb_secret_...`、または旧 service_role `eyJ...`）
   - `GCV_API_KEY` ＝ Google Cloud Vision のАPIキー（車検証OCRを使う場合のみ。未設定でも端末側OCRに自動で切り替わります）
3. 保存後、「Deployments」→ 最新のデプロイの「⋯」→「Retry deployment」で**再デプロイ**（環境変数を反映させるため）。

---

## フェーズ3：動作確認

1. 公開URLを開く →「入庫受付」を選び、担当スタッフを選んでスタート。
2. 適当に入力し、5ページ目まで進む。
3. 「保存番号」に短いID（例：a1b2c3d4）が表示され、「データベースへ反映されました」と出ればOK。
4. Supabaseの「Table editor」→「sheets」に行が増えていることを確認。
5. 「共有QRを作成」→ スマホでQRを読み取り、`view.html?id=...` が開いて受付票が表示されればQR共有も成功。
6. 開始画面の「1ヶ月以内／3ヶ月以内」に、保存した見積もり（種別＝見積もり依頼で保存したもの）が一覧表示されることを確認。

---

## うまくいかないとき
- 保存番号が「ローカルの番号」のまま／一覧が出ない → 環境変数が未設定、または再デプロイしていない可能性。フェーズ2の環境変数→再デプロイを確認。
- 503エラー（not_configured）→ 同上（SUPABASE_URL / SUPABASE_SERVICE_KEY 未設定）。
- 車検証OCRがサーバーで動かない → `GCV_API_KEY` 未設定でも端末側OCRで動作します（精度は控えめ）。

---

## セキュリティ・注意
- service_role キーはCloudflareの環境変数にのみ入力。GitHubやチャットには載せないでください。
- 共有データには個人情報（氏名・住所・写真・署名など）が含まれます。共有URLは推測されにくいIDですが、取り扱いにご注意ください（必要なら後日、有効期限や削除機能を追加できます）。
