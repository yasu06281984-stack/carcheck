# 入庫受付フォーム（車検証スキャン＋車両損傷チェック＋PDF）

お客様情報・車検証スキャン・損傷チェックが1枚にまとまった受付フォームです。
「印刷・PDF」を押すと、**お客様情報がPDF上部に自動で反映**されます。

## 画面の流れ
1. お名前・電話・住所を入力（メールは任意）。
2. 「車検証をスキャン」で写真を選ぶ → 車台番号・初年度年式・類別区分番号・型式指定番号・原動機の型式が自動入力（読めない欄は直接入力）。
3. 車両損傷チェックで、車種・ビューを切り替え、傷／凹み／その他を選んで図をタップ。部位は自動判定（違えばプルダウンで修正）、サイズ・備考・お預かり時の修理内容を入力。
4. 入庫時画像（任意）：前後を斜め45度から撮影推奨。通常2枚／VIP10枚まで。
5. PDFは3種類から選べます。
   - PDF（入庫時画像2枚あり）：お客様情報＋5面の損傷図＋一覧表＋入庫時画像2枚
   - PDF（画像なし）：お客様情報＋一覧表のみ（軽量・文字中心）
   - PDF（画像全て別紙）：1枚目にお客様情報＋一覧表、別紙に損傷図と入庫時画像すべて
   送信先を「PDFとして保存」。

## PDFの中身
- 上部：お客様情報（お名前・住所・電話＋車台番号・初年度登録・型式指定番号・類別区分番号・原動機の型式・受付日時）
- 中段：選択中の車種の5面（正面・後方・左右・ルーフ）に傷箇所マーカーを重ねた図
- 下段：入力チェック欄一覧／今回の修理希望箇所一覧（表）

## 公開（Cloudflare Pages）
1. このフォルダ一式を Cloudflare Pages にアップロード（フォルダをドラッグ＆ドロップ、またはGit連携）。
2. `functions/api/ocr.js` は自動で `/api/ocr` として動きます（設定不要）。
3. Pages の「設定 → 環境変数」に **`GCV_API_KEY`**（Google Cloud Vision の APIキー）を追加。
   - これが高精度OCR（サーバー側）に使われます。
4. 公開URLにアクセスして動作確認。

## ローカルで試す
- `index.html` をダブルクリックで開けば、入力・損傷チェック・PDFはそのまま動きます。
- ローカル（file://）では `/api/ocr` が無いため、車検証スキャンは**ブラウザ内OCR（Tesseract）に自動で切替**わります（精度は控えめ）。サーバー公開後はサーバー側OCRになります。

## カスタマイズ
- 屋号：`index.html` の「AUTO SHOP NAME / ○○自動車」を書き換え。
- 修理内容の選択肢や判定枠（部位範囲）は後から調整できます。
- 判定枠（修理箇所）の精度の最終調整は、全機能が固まってからまとめて実施予定。
